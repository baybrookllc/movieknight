# Movieknight Codebase Review Report - May 24, 2026

## Executive Summary
This report summarizes the findings of a comprehensive codebase review of the Movieknight project. The review focused on identifying bugs, security vulnerabilities, and optimization opportunities using static analysis and manual inspection.

## 1. Bugs & Code Quality
### React Hooks & State Management
- **Synchronous `setState` in `useEffect`:** Components such as `FriendsClient.tsx`, `MessagesClient.tsx`, and `NotificationsClient.tsx` call `setState` synchronously within `useEffect` (e.g., `setLoading(false)`). This practice triggers cascading re-renders, which can significantly impact UI performance.
- **Declaration Order & Missing Dependencies:** 
    - `components/AuthProvider.tsx`: `loadUserProfile` is called inside `useEffect` before it is declared.
    - `components/BrowseClient.tsx`: `fetchTriggersForResults` is accessed before declaration.
    - These issues, combined with missing dependencies in `useCallback` and `useEffect` hooks, can lead to stale closures and unpredictable UI behavior.

### Type Safety
- **Extensive Use of `any`:** There are over 100 instances of `@typescript-eslint/no-explicit-any` across core files (e.g., `mcp-server/src/index.ts`, `components/MessagesClient.tsx`). This undermines the benefits of TypeScript and increases the risk of runtime errors.

## 2. Security Flaws
### Rate Limiter "Fail Open"
- **File:** `supabase/functions/_shared/rate-limit.ts`
- **Issue:** The rate limiter is designed to "fail open" if Upstash Redis environment variables are missing. While helpful for local development, it creates a risk where a production misconfiguration would leave expensive AI endpoints (OpenAI/Anthropic) unprotected from abuse.

### Aggressive Telemetry & Data Leakage Risk
- **File:** `lib/debug-logger.ts`
- **Issue:** The logger intercepts all `console` methods and `fetch` requests globally. Without strict sanitization, this increases the risk of accidentally logging sensitive user data, PII, or auth headers into the database.

## 3. Optimization Opportunities
### Middleware Latency
- **File:** `proxy.ts`
- **Observation:** Fresh CSP generation and Supabase auth checks on every request add latency. Refining the middleware `matcher` to more aggressively exclude static assets and non-critical public routes would improve Time-to-First-Byte (TTFB).

### Database Query Efficiency
- **Observation:** Filtering with large arrays in `.in('id', [...])` REST calls can exceed URL length limits. Offloading these operations to Postgres RPC functions is more robust and efficient for large datasets.

### Client-Side Caching Performance
- **File:** `components/BrowseClient.tsx`
- **Observation:** Synchronous `JSON.parse` of large datasets from `sessionStorage` on mount can cause UI stuttering. Consider moving to an asynchronous store or a dedicated state management library for these caches.
